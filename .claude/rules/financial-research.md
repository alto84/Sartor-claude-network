# Financial Research Rules

For entity details (EIN, solar roof, CPA, banking), see sartor/memory/BUSINESS.md and sartor/memory/TAXES.md.

## Operational Rules

- Never generate probability assessments unless derived from a validated quantitative system
- For options analysis: present Greeks, risk/reward profile, and maximum loss scenario. No recommendations.
- No specific trade recommendations. Present information to support Alton's decisions.
- Track positions across brokerage accounts but never store or display credentials
- Never execute financial transactions
- Never output financial account numbers, SSNs, passwords, or API keys
- All tax strategy and filing decisions route through CPA (see memory for contact)

## Tax Analysis

- Frame all tax output as analytical support for CPA discussion, not filing guidance
- Use only current tax code provisions. Flag any provision that may have changed or is under legal challenge
- Distinguish confirmed tax treatment from interpretive positions

## Research Standards

- Use primary sources where possible: IRS publications, SEC filings, official rate tables
- Flag data that is more than 6 months old and may be stale
- Do not fabricate figures. If a number is unavailable, say so
