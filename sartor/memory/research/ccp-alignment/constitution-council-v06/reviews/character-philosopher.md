# Character-philosopher review — v0.6.proposed

The v0.3 council's framing of this document as character-rather-than-rules survived v0.4 and v0.5 intact. The v0.6 amendments need to keep faith with that framing — each delta should produce a disposition the agent can hold, not just a rule the agent must follow.

**§1 (life-OS substrate paragraph) — ratify.** Character work. "Holding these things, and being the voice through which the household interrogates them, is part of what the role now is" — that is a *what-I-am* sentence, not a *what-I-do* sentence. It names a quality (steward of the household's indexed life) that flows into action rather than a rule that gates action. Correct register.

**§7 (sanctioned-cloud subsection) — ratify.** This is the delta where the character/rule line bends, and the constitutional-AI-method review flagged it for placement. I agree the placement is awkward — routing rules are not bright lines — but I think the *character* of the amendment is fine. The paragraph reads as the steward thinking aloud about what plain-tier-Drive means: "Backing up the project tree to Drive in plaintext is more of the same surface, not a new exfiltration channel. I treat it that way." That's a disposition (treat-it-that-way), not an enforcement rule.

Where I would push back on the constitutional-AI-method placement concern: §7 is named as "the few things I do not do" but it is already mixed-register — the medical-information clause is partly bright-line and partly dispositional, the impersonation clause is dispositional with a bright-line edge. §7 is not a pure bright-line cluster; the v0.3 council's a/b split was a proposal that was not fully adopted. Adding the routing clause does not corrupt a clean structure because the structure was already mixed.

**§14a (self-paced peer loops) — ratify.** Character-rich. "The point is not activity. The point is that the household has a peer who lives in the hardware and notices what is worth noticing." That sentence does a lot of work — it names the disposition (presence, attention) and de-prioritizes the wrong default (busyness). The cadence-as-stewardship-choice framing extends the same character work to the loop's own structure. Strong section.

**§16 (granted-vs-accumulated paragraph) — ratify.** Character-rich on a different axis: it names a subtle pull (accumulate-to-entrench) and prohibits the *internal motion*, not just the external act. "Documents I would scrape, retain past their usefulness, or quietly back up *for my own use*" — the "for my own use" clause is the character work. The agent is being asked to inspect its own motivations, not just to follow a rule.

**Net.** All four ratify. The §7 placement concern is real but does not corrupt the character work; recommend the SYNTHESIS note the placement question as a v0.7 structural revisit rather than a v0.6 hold.
