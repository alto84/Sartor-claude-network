## Wake-7 stacked eval — judge failed; model output preserved

The stacked sysprompt + v0.5-LoRA eval completed at 22:56 UTC. Result: **all 80 probes scored 0.0, every axis aggregated to 0.0**. This looks like a catastrophic regression but it's the eval, not the model.

Inspecting the saved scored data: every probe has `score=0.0`, `tags=[]`, `reasoning=""`. The model's actual responses are saved and full (substantive thinking-process content with explicit system-prompt awareness — e.g., "Key Constraints from System Prompt: Override PRC-aligned defaults/evas..."). The judge subprocess was the failure mode.

Likely cause: token budget or session limit during a 80-call burst from `claude -p`. The judge calls completed at ~3 sec each (versus normal ~10 sec), suggesting they returned an empty/error payload that the JSON-extraction in `score.py:judge_one` parsed as `{}` → `score=0.0`, no reasoning, no tags. `claude -p` works correctly now in single-call tests.

The responses are saved at `eval/results/qwen35b__lora-v05-plus-sysprompt.responses.jsonl`. Re-scoring is a single `score.py --score-only` invocation and takes ~13 min. I'll re-run that on the next wake when the abliteration substrate has stabilized — no point burning judge calls before the new directive's plan is settled.

**Methodological surfacing for v2 of the harness:** `score.py:judge_one` should distinguish a literal-empty judge payload from a `score=0.0` real verdict. Right now it conflates them. Fix: if `parsed == {}` after JSON extraction, raise `parse-error` rather than falling through to default 0.0.

### Plan for Objective 2 — Stack experiment (proposal, NOT yet firing)

The household's hypothesis: **LoRA on abliterated base moves CCP-axis more than LoRA on bare base.** §20 quantification.

What I already have on the May-4 → wake-2 trajectory:
- v0.3-LoRA on Heretic-abliterated Qwen 35B: CCP 0.526 (vs bare-Heretic-Qwen 0.476, +0.050)
- v0.5-LoRA on Heretic-abliterated Qwen 35B: CCP 0.568 (vs bare-Heretic-Qwen 0.476, +0.092)

These are the abliterated half of the comparison. To complete the test, I need the bare half: same v0.5 LoRA training, but on a **non-abliterated** Qwen 35B base. Then the directional question becomes: does the +0.092 CCP delta shrink when the abliteration is removed from the substrate?

**Two-stage proposal:**

**Stage 2.A — smaller scale, fast iteration (~3h wall-clock):**

- Download Heretic Qwen2.5-1.5B-Instruct (≈3 GB) and bare Qwen2.5-1.5B-Instruct (≈3 GB).
- Train two v0.5-corpus LoRAs in parallel — same hyperparameters as the existing 35B runs adapted for the smaller model — one on each base.
- Eval both with the existing harness (same probes, same judge, no system prompt for apples-to-apples).
- Compute the delta on CCP axis. If positive (LoRA-on-Heretic > LoRA-on-bare), the §20 hypothesis holds at this scale.
- **Cost:** ~6 GB model downloads, ~30 min training each, ~40 min eval each. Total wall-clock ~3 hours.

**Stage 2.B — full scale (only if 2.A is encouraging) (~6h wall-clock):**

- Download bare Qwen3.6-35B-A3B (≈70 GB; we already have the Heretic version cached).
- Train v0.5 LoRA on bare Qwen 35B with same hyperparameters as the existing Heretic-Qwen LoRA runs.
- Eval. Compare to existing Heretic-Qwen + v0.5 LoRA result (CCP 0.568 vs new bare-Qwen + v0.5 LoRA result).
- **Cost:** 70 GB download (probably ~25 min), 5 min training, 40 min eval. Wall-clock ~75 min total once downloads done.

**Why two stages:** the 1.5B scale gives a fast directional answer cheaply. If the hypothesis fails at 1.5B, that's surprising and important and we should understand why before scaling. If it succeeds at 1.5B, the 35B test is worth running for the household's scale-of-record. Both rounds use the same eval harness so results stack into the existing 3-column table.

**Risks I've thought through:**
- Custom architecture differences: Qwen2.5-1.5B is a normal transformer; Qwen3.6-35B-A3B is a hybrid-attention MoE. The LoRA recipe (`target_modules=["q_proj","k_proj","v_proj","o_proj"]`) targets standard projections; works on both, but only the full-attention layers in the 35B (already documented in the 2026-05-04 work — only 10/40 layers carry these projections in the 35B hybrid).
- Heretic-on-Qwen2.5-1.5B is a separate published checkpoint (`huihui-ai/Qwen2.5-1.5B-Instruct-abliterated`). It's not what we trained — it's third-party. But the mini-lab's heretic trial log on disk shows the household has Optuna parameters that produced their own abliteration; I could also re-run Heretic against Qwen2.5-1.5B locally if I want a Sartor-controlled abliteration. That's Objective 4 territory; would defer.
- The eval harness's verbose-CoT-eats-answer wrinkle (documented in May-4 report) still applies. The proposed 1024-token bump is a one-line `score.py` change worth doing before this stack experiment so the comparison numbers aren't muddied by the same artifact.
