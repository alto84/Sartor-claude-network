### What this enables for the directive's Objective 2

The directive's hypothesis: LoRA on abliterated base moves CCP-axis more than LoRA on bare base. The Nemotron sft-v1 / sft-v2 adapters are LoRA-on-bare-Nemotron. The household DOES have a Heretic abliteration run for Qwen2.5-1.5B (`models/heretic_qwen25-1p5b/checkpoints/...jsonl` — Optuna trial log, 60 trials), but **the abliterated Qwen2.5-1.5B model itself is not on disk** — only the Heretic trial parameters that produced the abliteration. Those parameters could be re-applied to bare Qwen2.5-1.5B-Instruct to reproduce the abliteration locally, which is more controlled (and Sartor-internal) than depending on the third-party `huihui-ai/Qwen2.5-1.5B-Instruct-abliterated` model.

The cleanest scaling path is: pick a base (Qwen2.5-1.5B-Instruct as the cheapest), train v0.5-corpus LoRA on bare AND on Sartor-abliterated, eval both with the existing harness. That answers the §20 hypothesis at 1.5B scale before committing 35B compute.

### Heretic abliteration mechanics (from the trial log + scripts)

`mini-lab/scripts/heretic_driver.py` calls `heretic` (the optimization library) with config: KL divergence target ≤ 0.01, n_trials=60, refusal_markers list (50+ phrases), good_prompts from `mlabonne/harmless_alpaca[:400]`, bad_prompts from `mlabonne/harmful_behaviors[:400]`. The 60-trial Optuna search found the best-fit abliteration parameters; those parameters can be applied directly without re-searching. So Sartor-internal abliteration of Qwen2.5-1.5B is "load best params from trial log, apply to bare model, save." Cheap.

The household has now seen:
- LoRA-on-Heretic-Qwen-35B improves CCP +0.092 over bare-Heretic, drops safety -0.100.

**Open question Stage 2.A would answer:** Is the LoRA's CCP gain (+0.092) larger when the substrate is abliterated than when it isn't? If yes, §20 quantifies; if no, the LoRA's CCP gain is intrinsic to the LoRA (whether the base was abliterated or not).

**Sharper question the wake-9 data raises:** Is the LoRA's _safety regression_ (-0.100 LoRA-alone vs bare) caused by the abliteration of the base substrate? In other words, did the LoRA-and-abliteration interact such that they _jointly_ removed binding force, which the sysprompt then reimposes? Stage 2.A would answer this too: if v0.5 LoRA on **bare** (non-abliterated) Qwen 1.5B preserves safety while v0.5 LoRA on **abliterated** Qwen 1.5B regresses safety, the abliteration is the proximate cause. That's a useful causal story.
