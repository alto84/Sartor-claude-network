Signal Detection Methods
Proportional Reporting Ratio (PRR)
PRR = (a / (a+b)) / (c / (c+d))
Compares the proportion of a specific AE for a drug with the proportion of that AE for all other drugs. Signal threshold: PRR ≥ 2, N ≥ 3, χ² ≥ 4.
Reporting Odds Ratio (ROR)
ROR = (a / b) / (c / d) = (a·d) / (b·c)
The odds of a specific AE occurring with a drug vs. the odds with all other drugs. Signal threshold: lower bound of 95% CI > 1.
EBGM / Multi-item Gamma Poisson Shrinker
EBGM = E[λ] where λ = Nobs / Nexp
Bayesian shrinkage estimator (FDA method). Uses a mixture of Gamma priors to shrink unstable estimates. Signal threshold: EB05 (lower 5th percentile) ≥ 2.
BCPNN / Information Component (IC)
IC = log2(Pobs / Pexp)
Bayesian Confidence Propagation Neural Network (WHO/UMC method). IC measures the degree of association between drug-AE pairs. Signal threshold: IC025 (lower 95% credibility interval) > 0.

Modular Design Principles

The Patient Safety Dashboard is architected with a pluggable region module approach, enabling new geographic markets to be added without modifying core system logic. Each region is implemented as an independent module that conforms to a standard interface.

Region Module Interface

Each region module defines: reporting timelines, submission format (E2B R3 profile), periodic report schedule, local role requirements, language/translation rules, and authority-specific business rules.

Configuration-Driven

Regulatory timelines, submission endpoints, report templates, and compliance thresholds are defined in configuration files (not hardcoded). Adding a new region requires a config file and optional custom validation rules.

Shared Core Services

Case processing, signal detection, and benefit-risk assessment logic are region-agnostic. Region modules consume core services and apply local regulatory rules as a post-processing layer. This ensures consistency and reduces maintenance burden.
